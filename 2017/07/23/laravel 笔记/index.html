第一章：laravel环境搭建及目录基本信息

声明：此次适用于laravel 5.4 版本

1.安装composeer
要求：php开启openssl拓展

如果是wamp环境，apache需要开启rewrite模块

php版本需高于5.6

2.下载composer

安装使用：需要指定php的exe文件路径
下一步使用composer部署laravel

国内使用composer管理依赖一般会先切换源地址为国内镜像，代码如下

composer  config -g repo.packagist composer https://packagist.phpcomposer.com

打开cmd；输入下面一行代码

composer create-project laravel/laravel --prefer-dist

[project_name] ：项目名称、

–[5.3.*] laravel版本号

稍等片刻之后，laravel就会在你的当前目录部署完成

ps：系统文件夹下似乎不能创建项目

3.启动laravel

可使用php自带的服务器开启测试，简单方便，缺点是，每次开启都需要运行一次命令,在你的项目目录下打开命令行，运行如下代码

php artisan serve

会在本地端口8000（默认）启动laravel。浏览器地址栏输入 localhost:8000
即可访问laravel

5.laravel目录结构分析
详情去看laravel中文文档

第二章 入门及使用

1.定义路由配置文件

路由文件的位置;laravel/routes/web.php

格式：Route::请求方式（’请求的url’，闭包函数或相应的方法）
例：

Route::get('/',function(){return 'hello world'})

函数的返回值就是请求的响应

–请求方式：get post |前者常用| put patch delete options

–match和any方法：match匹配所选请求方法，any对应所有请求方法

默认根路由：view的位置在laravel/routes/views/welcome.blade.php

路由传参：Route::get(url/{参数？}，function($参数){});

？表示该参数可有可无，需定义默认值

–[->where(‘参数’,regex)]表示对参数匹配正则，不匹配时会导向错误页面，多个参数正则匹配格式为where（[‘参数1’=>’正则1’,’参数2’=>’正则2’])

路由别名，可以在其他程序获取他的路由地址，

Route::get('/user/login',['as'=>'name',fubction(){
    return route('name');
}]);

路由前缀格式：

Route::group(['prefix'=>'admin'],function(){
    route::get('login',function(){});
    route::get('logout',function(){});
});

其等同于

Route::get('/admin/login',function(){});
Route::get('admin/logout',function(){});

控制器路由：

Route::get('url','Controller@method');

  1. ###六 CSRF攻击 ###
    Cross-site request forgery 跨域请求伪造 攻击

laravel中避免csrf攻击的策略为：每个用户session生成一个CSRF Token，该Token用于验证登陆用户和发起请求是否是同一个人，如果不是则请求失败。

在视图文件中加入 

{{csrf_field()}}

,该函数生成一个hidde的text域
也可以添加 

{{csrf_token()}}

直接生成token字符串。

并不是所有的请求都需要避免csrf攻击，比如第三方API获取数据的请求。
可以在VerifyCsrfToken (app/Http/Middleware/VerifyCsrfToken.php)中讲要排除的请求URL添加到$except 中,例：

prtected $except=[
    'demo/registerok'    
];

七 Controller

创建模型：

php artisan make:model Admin
  • Admin:model 名
  • 该命令应该在项目根目录执行
  • 生成的模型类文件默认存放在app/http/Models下

model类文件中应
注意：

  • $table属性,值是不要前缀的表名，不指定则默认为类名的复数s作为表名
  • $primaryKey
  • $timestamps 值为false，不设置为false则默认会操作表中created_at和updated_at字段
  • $fillable属性，表示提交表单时，允许修改到数据库的字段信息。

创建相应的控制器：

php artisan make:cntroller Admin\AdminController

对数据库的操作：

  1. 定义路由

  2. 编写方法

    添加数据： ①实例化模型，然后为模型设置属性，最后使用save方法即可

    ②引用request类（ Illuminate\Http\Request），使用模型的create方法添加

@method 1: 
$admin=new Admin();
$admin->name='name';
$admin->age=18;
$admin->email='admin@live.com';
$res=$admin->save();


@method 2；
$data=Request::all();
$res=Admin::create($data)

Request类的方法还有input,only,except,has,get等。想要了解，去看文档~

查询数据：使用模型的find（），all()静态方法，还有where条件语句,详细查手册

修改数据： laravel更新数据，需要先调用find找到对应记录，然后反正的对象进行更新数据，最后调用save（）方法保存即可。

$user=User::find($id);
$user->title=$_POST['title'];
$user->name=$_POST['name'];
$user-<save();

删除数据：同样需要先查询对应数据，然后使用返回对象的delete方法来删除。

$user=User::find($id);
$user->delete();

复杂查询：链式查询，关键字：where,select,orderby,get,take,offset等，详情查看手册

自动验证

laravel验证方式有两种：控制器里写验证规则，单独写一个验证文件

①控制器里写验证,例：

public function store(Request $request){
$this->validate($request,[
    'name'=>'qequired|unique:posts|max:25',
    'age'=>'reqqired',
]);

}

基本验证规则有：

  • required: 不能为空
  • max:255最长255个字符，
  • min:1最少1个字符
  • email:验证邮箱是否合法
  • confirmed:验证两个字段是否相同，如果验证的字段是password,
    则必须输入一个与之匹配的password_confirmation字段
  • integer:验证字段必须是整型
  • ip:验证字段必须是IP地址
  • numeric 验证字段必须是数值
  • max:value 验证字段必须小于等于最大值，和字符串，数值，文件字段的size规则一起使用。
  • min:value 验证字段的最小值，对字符串、数值、文件字段而言，和size规则使用方式一致。
  • size:value 验证字段必须有和给定值value想匹配的尺寸，对字符串而言，value是相应的字符数目，对数值而言，value是给定整型值；对文件而言，value是相应的文件字节数。
  • string 验证字段必须是字符串
  • unique:表名，字段，需要排除的ID
